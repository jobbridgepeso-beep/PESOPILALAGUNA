# Requirements Document

## Introduction

JobBridge is an Intelligent Chatbot and OCR-Assisted Job Matching and Employment Monitoring System for the Public Employment Service Office (PESO) of Pila, Laguna. The system digitizes and automates PESO Pila's core employment operations, including job posting and matching, applicant management, program enrollment (SPES, DILP, OWWA, Manpower Skills Training), job fair management with QR attendance, referral letter generation, Labor Market Information (LMI) reporting, and real-time notifications. The platform serves four user roles — Jobseeker, Employer, PESO Staff, and Admin — each with a dedicated module set and access-controlled routing.

---

## Glossary

- **JobBridge**: The web-based employment management platform for PESO Pila, Laguna.
- **PESO**: Public Employment Service Office — the government agency managing local employment services.
- **Jobseeker**: A registered user seeking employment through the platform.
- **Employer**: A registered organization posting job vacancies and managing applicants.
- **PESO_Staff**: A PESO employee managing operations, programs, and reports. Account created by Admin only.
- **Admin**: The highest-privilege user, pre-seeded in the database, managing staff accounts, audit trails, and system settings.
- **AI_Matcher**: The job matching engine using TF-IDF Vectorizer and Cosine Similarity (scikit-learn) to compute match scores between jobseeker profiles and job vacancies.
- **OCR_Processor**: The Google Vision API + spaCy NLP pipeline that extracts structured data from uploaded resume documents.
- **Chatbot**: The Dialogflow ES-powered floating assistant available to Jobseekers for guided navigation and FAQ responses.
- **QR_Scanner**: The QR code generation and scanning subsystem used for job fair and training attendance.
- **Notification_Service**: The real-time notification system using Socket.io and Flask-Mail (Gmail SMTP) for in-app and email notifications.
- **LMI_Report**: Labor Market Information report generated monthly, quarterly, and annually by the APScheduler.
- **OTP**: One-time password — a 6-digit code with 10-minute expiry used for email verification during registration and password reset.
- **JWT**: JSON Web Token — the authentication token with a 15-minute access token lifetime and a refresh token stored in an httpOnly cookie.
- **Referral_Letter**: A PDF document generated by PESO_Staff for a Jobseeker applying to a specific vacancy.
- **SPES**: Special Program for Employment of Students — a PESO-managed youth employment program.
- **DILP**: Displacement/Livelihood Program — a PESO-managed livelihood assistance program.
- **OWWA**: Overseas Workers Welfare Administration — a PESO-managed program for overseas workers.
- **Supabase**: The PostgreSQL-based backend-as-a-service providing the database, realtime subscriptions, and file storage.
- **Redis**: In-memory data store used for caching, rate limiting, and Celery task queuing.
- **Audit_Trail**: An append-only, immutable log of all significant system actions, accessible only by Admin.
- **ProtectedRoute**: A React Router v6 component that enforces authentication before rendering a route.
- **RoleGuard**: A React Router v6 component that enforces role-based access control on protected routes.
- **WeasyPrint**: The PDF generation library used for referral letters, LMI reports, and certificates.
- **APScheduler**: The Python scheduler that triggers automated LMI report generation.

## Requirements

---

### Requirement 1: User Registration and Email OTP Verification

**User Story:** As a Jobseeker or Employer, I want to self-register and verify my email with an OTP, so that I can access the platform with a verified account.

#### Acceptance Criteria

1. WHEN a Jobseeker or Employer submits a registration form with valid required fields, THE JobBridge SHALL create a pending account and send a 6-digit OTP to the provided email address via Gmail SMTP.
2. WHEN an OTP is submitted within 10 minutes of issuance, THE JobBridge SHALL activate the account and redirect the user to their role-specific dashboard.
3. IF an OTP is submitted after its 10-minute expiry, THEN THE JobBridge SHALL reject the OTP and prompt the user to request a new one.
4. IF an OTP is submitted that has already been used, THEN THE JobBridge SHALL reject the submission and return an error message.
5. WHEN a user requests OTP resend, THE JobBridge SHALL invalidate the previous OTP and issue a new 6-digit OTP with a fresh 10-minute expiry.
6. THE JobBridge SHALL enforce rate limiting on the /register endpoint to prevent abuse.

---

### Requirement 2: Authentication and Session Management

**User Story:** As any registered user, I want to log in securely and have my session managed automatically, so that my account remains protected.

#### Acceptance Criteria

1. WHEN a user submits valid credentials, THE JobBridge SHALL issue a JWT access token with a 15-minute lifetime and store a refresh token in an httpOnly cookie.
2. WHEN a JWT access token expires, THE JobBridge SHALL automatically use the refresh token to issue a new access token without requiring the user to log in again.
3. IF a login attempt fails 5 times from the same IP address within 15 minutes, THEN THE JobBridge SHALL block further login attempts from that IP for the remainder of the 15-minute window.
4. WHEN a user is idle beyond the role-specific session timeout threshold, THE JobBridge SHALL invalidate the session and redirect the user to the login page.
5. WHEN a user logs out, THE JobBridge SHALL invalidate the refresh token and clear the httpOnly cookie.
6. THE JobBridge SHALL hash all passwords using bcrypt with a cost factor of 12 before storing them in the database.

---

### Requirement 3: Role-Based Access Control and Routing

**User Story:** As any user, I want to access only the modules and routes permitted for my role, so that unauthorized access to other roles' data is prevented.

#### Acceptance Criteria

1. THE JobBridge SHALL enforce ProtectedRoute components on all authenticated routes, redirecting unauthenticated users to the login page.
2. THE JobBridge SHALL enforce RoleGuard components on all role-specific routes, returning a 403 response for users accessing routes outside their assigned role.
3. THE JobBridge SHALL route Jobseeker modules under the /jobseeker/* path prefix.
4. THE JobBridge SHALL route Employer modules under the /employer/* path prefix.
5. THE JobBridge SHALL route PESO_Staff modules under the /staff/* path prefix.
6. THE JobBridge SHALL route Admin modules under the /admin/* path prefix.
7. WHEN an Admin creates a PESO_Staff account, THE JobBridge SHALL send the login credentials to the staff member's email and require a password change on first login.
8. THE Admin account SHALL be pre-seeded in the database with no self-registration pathway.

---

### Requirement 4: Jobseeker Profile and OCR Resume Parsing

**User Story:** As a Jobseeker, I want to upload my resume and have my profile auto-populated via OCR, so that I can quickly build an accurate profile without manual data entry.

#### Acceptance Criteria

1. WHEN a Jobseeker uploads a resume file in JPG, PNG, or PDF format not exceeding 5MB, THE OCR_Processor SHALL extract structured data including name, contact information, education, work experience, and skills.
2. WHEN OCR extraction is complete, THE JobBridge SHALL pre-populate the Jobseeker's profile fields with the extracted data for review and confirmation.
3. IF a Jobseeker uploads a file exceeding 5MB or in an unsupported format, THEN THE JobBridge SHALL reject the upload and return a descriptive error message.
4. WHEN a Jobseeker saves their profile, THE JobBridge SHALL persist all profile fields to the database.
5. THE JobBridge SHALL allow Jobseekers to manually edit any OCR-extracted field before saving.

---

### Requirement 5: Employer Profile and Company Profile Management

**User Story:** As an Employer, I want to manage my account and company profile, so that Jobseekers and PESO Staff can view accurate company information.

#### Acceptance Criteria

1. WHEN an Employer completes registration and email OTP verification, THE JobBridge SHALL create an Employer account and an associated Company Profile record.
2. THE JobBridge SHALL allow an Employer to update company name, address, industry, contact details, and company description at any time.
3. WHEN an Employer updates the Company Profile, THE JobBridge SHALL persist the changes and reflect them on all associated job vacancy listings.

---

### Requirement 6: Job Vacancy Management

**User Story:** As an Employer, I want to post and manage job vacancies, so that qualified Jobseekers can discover and apply for open positions.

#### Acceptance Criteria

1. WHEN an Employer submits a new job vacancy with all required fields, THE JobBridge SHALL create the vacancy in a pending state awaiting PESO_Staff approval.
2. WHEN a PESO_Staff member approves a job vacancy, THE JobBridge SHALL publish the vacancy and make it visible in Jobseeker job search results.
3. IF a PESO_Staff member rejects a job vacancy, THEN THE JobBridge SHALL notify the Employer with the rejection reason and set the vacancy status to rejected.
4. WHEN an Employer closes a vacancy, THE JobBridge SHALL remove it from active search results and prevent new applications.
5. THE JobBridge SHALL allow PESO_Staff to manage all job vacancies including editing, approving, rejecting, and closing vacancies.

---

### Requirement 7: AI-Powered Job Matching

**User Story:** As a Jobseeker, I want to see a match score for each job vacancy based on my profile, so that I can prioritize applications for positions I am most qualified for.

#### Acceptance Criteria

1. WHEN a Jobseeker views job search results, THE AI_Matcher SHALL compute a cosine similarity score between the Jobseeker's skills and experience and each active vacancy's requirements using TF-IDF vectorization.
2. THE JobBridge SHALL display the computed match score alongside each vacancy in the Jobseeker's job search results.
3. WHEN an Employer views applicants for a vacancy, THE AI_Matcher SHALL compute and display a match score for each applicant relative to the vacancy requirements.
4. THE JobBridge SHALL sort job search results by match score in descending order by default.

---

### Requirement 8: Job Application Workflow

**User Story:** As a Jobseeker, I want to apply for job vacancies and track my application status in real time, so that I stay informed throughout the hiring process.

#### Acceptance Criteria

1. WHEN a Jobseeker submits an application for an active vacancy, THE JobBridge SHALL create an application record and notify the Employer via the Notification_Service.
2. WHEN an Employer updates an application status (reviewed, shortlisted, rejected, hired), THE Notification_Service SHALL send a real-time in-app notification and an email to the Jobseeker.
3. WHEN a Jobseeker is marked as hired, THE JobBridge SHALL create an Employment Monitoring record linked to the Jobseeker, Employer, and vacancy.
4. WHEN a PESO_Staff member generates a Referral_Letter for an application, THE JobBridge SHALL produce a PDF document using WeasyPrint and make it available for the Jobseeker to download.
5. THE JobBridge SHALL display all application statuses and history on the Jobseeker's My Applications module with real-time updates via Socket.io.

---

### Requirement 9: Interview Management

**User Story:** As an Employer, I want to schedule and manage interviews for shortlisted applicants, so that the hiring process is organized and communicated clearly.

#### Acceptance Criteria

1. WHEN an Employer schedules an interview for a shortlisted applicant, THE Notification_Service SHALL send an email invitation and an in-app notification to the Jobseeker with the interview date, time, and location or link.
2. WHEN an interview is updated or cancelled, THE Notification_Service SHALL notify the affected Jobseeker immediately via email and in-app notification.
3. THE JobBridge SHALL display all scheduled interviews on the Jobseeker's Interview Schedule module.
4. THE JobBridge SHALL allow PESO_Staff to view and oversee all interview schedules across all employers via the Interview Oversight module.

---

### Requirement 10: Job Fair Management and QR Attendance

**User Story:** As a PESO_Staff member, I want to create job fair events and track attendance via QR codes, so that participation records are accurate and automated.

#### Acceptance Criteria

1. WHEN a PESO_Staff member creates a job fair event, THE JobBridge SHALL generate a unique QR code for each registered Jobseeker participant.
2. WHEN a Jobseeker registers for a job fair, THE JobBridge SHALL send the Jobseeker's unique QR code to their registered email address.
3. WHEN a PESO_Staff member scans a Jobseeker's QR code using the QR_Scanner on the event day, THE JobBridge SHALL record the attendance timestamp and update the attendance record in real time.
4. IF a QR code is scanned that does not correspond to a registered participant, THEN THE JobBridge SHALL display an error message on the scanner interface.
5. THE JobBridge SHALL allow Employers to participate in job fair events through the Employer Job Fair module.
6. THE JobBridge SHALL generate an attendance report exportable to Excel using OpenPyXL for each job fair event.

---

### Requirement 11: PESO Program Management (SPES, DILP, OWWA, Manpower Skills Training)

**User Story:** As a Jobseeker, I want to apply for PESO programs and upload required documents, so that I can access government employment assistance.

#### Acceptance Criteria

1. WHEN a Jobseeker submits a program application with required documents in JPG, PNG, or PDF format not exceeding 5MB each, THE OCR_Processor SHALL extract relevant data from the uploaded documents.
2. WHEN a PESO_Staff member reviews a program application, THE JobBridge SHALL display the OCR-extracted data alongside the original uploaded documents.
3. WHEN a PESO_Staff member approves or rejects a program application, THE Notification_Service SHALL notify the Jobseeker via email and in-app notification with the decision and reason.
4. THE JobBridge SHALL maintain separate management modules for SPES, DILP, OWWA, and Manpower Skills Training, each accessible to PESO_Staff.
5. WHEN a Manpower Skills Training or DILP session uses attendance tracking, THE QR_Scanner SHALL record participant attendance in the same manner as job fair attendance.
6. THE JobBridge SHALL allow PESO_Staff to export program enrollment and attendance data to Excel.

---

### Requirement 12: Employment Monitoring

**User Story:** As a PESO_Staff member, I want to monitor the employment status of placed Jobseekers, so that accurate employment data is maintained for reporting.

#### Acceptance Criteria

1. WHEN a Jobseeker is marked as hired through the application workflow, THE JobBridge SHALL automatically create an Employment Monitoring record with the Jobseeker, Employer, vacancy, and hire date.
2. THE JobBridge SHALL allow PESO_Staff to update employment status records including start date, end date, and employment type.
3. THE JobBridge SHALL display employment history on the Jobseeker's Employment Monitoring module.
4. THE JobBridge SHALL include employment monitoring data as a source for LMI report generation.

---

### Requirement 13: Labor Market Information (LMI) Reports

**User Story:** As a PESO_Staff member or Admin, I want automated LMI reports generated on a schedule, so that accurate labor market data is available without manual compilation.

#### Acceptance Criteria

1. THE APScheduler SHALL automatically generate LMI reports on a monthly, quarterly, and annual schedule using employment, application, and program data from the database.
2. WHEN an LMI report is generated, THE JobBridge SHALL produce a PDF version using WeasyPrint and an Excel version using OpenPyXL.
3. THE JobBridge SHALL make generated LMI reports available for download by PESO_Staff and Admin from the LMI Reports module.
4. THE JobBridge SHALL allow PESO_Staff and Admin to manually trigger LMI report generation for a specified date range.

---

### Requirement 14: Notifications System

**User Story:** As any user, I want to receive real-time in-app and email notifications for relevant events, so that I am always informed of actions that affect me.

#### Acceptance Criteria

1. WHEN a notification-triggering event occurs (application status change, interview scheduled, program decision, announcement), THE Notification_Service SHALL deliver an in-app notification via Socket.io to the affected user within 5 seconds.
2. WHEN a notification-triggering event occurs, THE Notification_Service SHALL send an email to the affected user's registered email address via Gmail SMTP.
3. THE JobBridge SHALL display unread notification counts and a notification list in each role's Notifications module.
4. WHEN a user marks a notification as read, THE JobBridge SHALL update the notification status and decrement the unread count in real time.
5. THE JobBridge SHALL allow PESO_Staff to create and send system-wide announcements to all Jobseekers or Employers via the Announcements module.

---

### Requirement 15: Chatbot Assistant

**User Story:** As a Jobseeker, I want a chatbot assistant available on the platform, so that I can get quick answers to common questions and navigate the system easily.

#### Acceptance Criteria

1. THE Chatbot SHALL be available as a floating button on all Jobseeker-facing pages.
2. WHEN a Jobseeker sends a message to the Chatbot, THE Chatbot SHALL process the message through Dialogflow ES and return a relevant response within 3 seconds.
3. THE Chatbot SHALL support intents for common queries including job search guidance, application status inquiries, program eligibility questions, and PESO office information.
4. IF the Chatbot cannot match a user message to a known intent, THEN THE Chatbot SHALL respond with a fallback message directing the user to contact PESO Staff.

---

### Requirement 16: Admin — Staff Account Management

**User Story:** As an Admin, I want to create and manage PESO Staff accounts, so that only authorized personnel can access staff-level operations.

#### Acceptance Criteria

1. WHEN an Admin creates a new PESO_Staff account with required fields, THE JobBridge SHALL create the account, generate a temporary password, and send the login credentials to the staff member's email.
2. WHEN a PESO_Staff member logs in for the first time, THE JobBridge SHALL require the staff member to change the temporary password before accessing any module.
3. THE JobBridge SHALL allow an Admin to deactivate or reactivate PESO_Staff accounts.
4. WHEN a PESO_Staff account is deactivated, THE JobBridge SHALL immediately invalidate all active sessions for that account.

---

### Requirement 17: Audit Trail

**User Story:** As an Admin, I want an immutable audit trail of all significant system actions, so that accountability and traceability are maintained across the platform.

#### Acceptance Criteria

1. THE JobBridge SHALL record an Audit_Trail entry for every significant action including login, logout, account creation, profile update, vacancy approval/rejection, application status change, program decision, report generation, and system settings change.
2. WHEN an Audit_Trail entry is created, THE JobBridge SHALL record the acting user's ID, role, action type, affected resource, timestamp, and IP address.
3. THE Audit_Trail SHALL be append-only; no user including Admin SHALL be able to modify or delete existing audit entries.
4. THE JobBridge SHALL allow Admin to search and filter the Audit_Trail by user, role, action type, date range, and affected resource.

---

### Requirement 18: System Settings

**User Story:** As an Admin, I want to manage system-wide configuration settings, so that platform behavior can be adjusted without code changes.

#### Acceptance Criteria

1. THE JobBridge SHALL provide an Admin-only System Settings module for configuring parameters including session timeout durations per role, rate limiting thresholds, OTP expiry duration, and file upload size limits.
2. WHEN an Admin saves updated system settings, THE JobBridge SHALL apply the new configuration values immediately without requiring a system restart.
3. THE JobBridge SHALL record all system settings changes in the Audit_Trail.

---

### Requirement 19: File Upload Security and Storage

**User Story:** As any user uploading files, I want the system to validate and securely store my uploads, so that malicious files cannot compromise the platform.

#### Acceptance Criteria

1. THE JobBridge SHALL accept only JPG, PNG, and PDF file types for all upload endpoints.
2. IF a user uploads a file exceeding 5MB, THEN THE JobBridge SHALL reject the upload and return a descriptive error message before storing any data.
3. THE JobBridge SHALL store all uploaded files in Supabase Storage with access controlled by the uploader's role and ownership.
4. THE JobBridge SHALL validate file MIME type on the server side in addition to file extension validation.

---

### Requirement 20: API Design and Error Handling

**User Story:** As a frontend developer, I want all API responses to follow a consistent format, so that error handling and data consumption are predictable across the application.

#### Acceptance Criteria

1. THE JobBridge SHALL return all API responses in the format `{ success: true/false, data: {...}, message: 'string' }`.
2. WHEN an API request succeeds, THE JobBridge SHALL set `success` to `true` and populate the `data` field with the response payload.
3. IF an API request fails due to validation, authentication, authorization, or server error, THEN THE JobBridge SHALL set `success` to `false`, set `data` to `null` or an empty object, and populate `message` with a descriptive error string.
4. THE JobBridge SHALL apply CORS restrictions to allow requests only from the configured Vercel frontend domain.

---

### Requirement 21: Forgot Password and Password Reset

**User Story:** As any registered user, I want to reset my password via email OTP, so that I can regain access to my account if I forget my password.

#### Acceptance Criteria

1. WHEN a user submits a forgot password request with a registered email address, THE JobBridge SHALL send a 6-digit OTP to that email address via Gmail SMTP.
2. WHEN a user submits a valid, unexpired OTP along with a new password, THE JobBridge SHALL update the password hash using bcrypt with cost factor 12 and invalidate the OTP.
3. IF the OTP submitted during password reset has expired or has already been used, THEN THE JobBridge SHALL reject the request and prompt the user to request a new OTP.
4. WHEN a password is successfully reset, THE JobBridge SHALL invalidate all existing active sessions for that account.

---

### Requirement 22: Dashboard and Analytics

**User Story:** As any user, I want a role-specific dashboard with relevant statistics and charts, so that I can quickly understand the current state of my activities.

#### Acceptance Criteria

1. WHEN a Jobseeker accesses the dashboard, THE JobBridge SHALL display statistics including total applications submitted, pending applications, interview schedules, and recommended job matches.
2. WHEN an Employer accesses the dashboard, THE JobBridge SHALL display statistics including active vacancies, total applicants, pending reviews, and hired count.
3. WHEN a PESO_Staff member accesses the dashboard, THE JobBridge SHALL display statistics including total registered jobseekers, active vacancies, pending approvals, upcoming job fairs, and program enrollment counts.
4. WHEN an Admin accesses the dashboard, THE JobBridge SHALL display all PESO_Staff statistics plus system health indicators including active user sessions and recent audit trail entries.
5. THE JobBridge SHALL render dashboard statistics using Chart.js or Recharts with responsive layouts.
